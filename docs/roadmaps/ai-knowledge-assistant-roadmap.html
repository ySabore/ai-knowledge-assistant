top-k=5 retrieval with MMR reranking
Prompt template enforcing citation format
Confidence heuristic from retrieval score

Phase 3 · Q3
Enterprise
Months 7–9
Milestone: First enterprise contract signed
SSO & SAML 2.0Scale▼
Okta, Azure AD, Google Workspace
Effort
What we're building
Enterprise SSO via SAML 2.0 and OIDC. Support Okta, Azure AD, Google Workspace as identity providers.
Definition of done
Enterprise IT can provision users via their existing IdP. No separate credential management.
Key tasks
SAML 2.0 + OIDC library integration
Okta, Azure AD, Google Workspace tested
JIT provisioning on first login
SCIM provisioning for automated deprovisioning
RBAC & permission syncScale▼
Per-doc ACLs, query-time filtering
Effort
What we're building
Role-based access control engine that mirrors document permissions from source systems. Users only see answers sourced from docs they can access.
Definition of done
Sarah in Eng sees Eng docs. Mike in Sales sees only Sales docs. Zero cross-team data leaks.
Key tasks
Permission propagation engine per connector
Per-document ACL storage in Postgres
Query-time permission filtering in retrieval step
Admin override capability with audit trail
Audit loggingScale▼
Immutable query + access trail
Effort
What we're building
Immutable log of all queries, document accesses, admin actions, and connector events. Exportable to SIEM.
Definition of done
Compliance teams can review who asked what, when, and what source it cited.
Key tasks
Append-only audit table in Postgres
Structured JSON log format with full context
Export to Splunk, Datadog, or S3
Configurable retention policy per org

Phase 4 · Q4
Moat
Months 10–12
Milestone: Series A readiness
Slack bot integrationMoat▼
/ask in channels, cited threaded replies
Effort
What we're building
Answer questions directly in Slack channels and DMs. /ask command or @mention triggers. Responses include citations in thread.
Definition of done
Teams get answers without leaving Slack. Massively increases DAU and stickiness.
Key tasks
Slack Bolt SDK, slash command + event subscriptions
Thread-aware multi-turn context handling
Admin controls per Slack channel
Rate limiting and abuse prevention
VPC / on-premise deployMoat▼
Docker + Helm chart, air-gap mode
Effort
What we're building
Packaged deployment option for customers who cannot use SaaS. Docker Compose and Helm chart for Kubernetes.
Definition of done
Unlocks regulated industries: finance, healthcare, government. Required for largest deals.
Key tasks
Docker Compose + Helm chart packaging
Swap cloud vector DB for self-hosted Qdrant
Swap managed DB for customer Postgres
Air-gap mode (no outbound except LLM API)
Fine-tuned retrieval modelMoat▼
Per-customer reranker, feedback loop
Effort
What we're building
Train a custom reranker on customer query-answer feedback to improve retrieval relevance over time. The core compounding moat.
Definition of done
Retrieval quality improves the longer a customer uses the product. Creates real switching cost.
Key tasks
Collect implicit feedback (thumbs, dwell, follow-ups)
Fine-tune BGE or Cohere cross-encoder reranker
A/B test reranker vs base retrieval per org
Customer-specific model versioning + rollback
Public API & webhooksMoat▼
Embed answer engine in any workflow
Effort
What we're building
Public REST API so enterprise customers can embed the answer engine in their own products and workflows.
Definition of done
Unlocks integration revenue. Customers build the product into internal portals and tools.
Key tasks
OpenAPI 3.0 spec + interactive docs
API key management with scoping per resource
Rate limiting per key with burst allowance
Webhook events: answer.created, sync.complete
